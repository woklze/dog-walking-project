# Требования к приложению:
#### Минимальные критерии работоспособности
Сборка и запуск
- [✓] Приложение собирается без ошибок.
- [+] Секреты не указаны в коде, выбран надежный способ их подтягивать. 

Есть README с:
- [+] требованиями (Java версия, Maven/Gradle, Docker)
- [+] запуск локально средствами Java или в контейнере
- [+] информацией, где хранить секреты
- [+] описанием контракта
- [+] портами и учетными данными для демо.
- [+] Поднята бд. 

Здоровье сервиса
- [+] Healthcheck приложения.
- [+] Отдельно виден статус соединения с БД или кидается понятная ошибка при недоступной БД.

Миграции и схема БД
- [+] бд заполняется приложением без ручного создания таблиц. Схема описана миграциями (Flyway/Liquibase) и накатывается автоматически при старте.
- [+] Есть внешние ключи/constraint там, где нужно по предметной области.

Сквозные пользовательские сценарии (2–3 ключевых)
- [+]  Для каждого сценария есть последовательность REST-запросов (создание → чтение → обновление/удаление).
- [+] Используются корректные методы и статусы: POST 201, GET 200, PUT/PATCH 200/204, DELETE 204.

Валидация и обработка ошибок
- [+]  Неверные данные → 400 с понятным телом ошибки.
- [+] Отсутствующие ресурсы → 404, невалидная аутентификация/доступ → 401/403 (если есть авторизация).
- [+] Нет 500 на предсказуемых ошибках пользователя.

Сохранность данных
- [+]  После перезапуска контейнера/приложения ранее созданные сущности сохраняются.

Логи и наблюдаемость
- [+] Логи содержат время, уровень, краткое описание события/ошибки.
- [+] Уровень логирования настраивается в конфигурации приложения.

Рекомендуемые (повышенный уровень, за дополнительные баллы)
- [+] Документация API: Swagger/OpenAPI доступен по /swagger-ui или есть актуальная Postman-коллекция с примерами.

Минимальные тесты:
- [ ] 3–5 unit-тестов ключевой бизнес-логики,
- [ ] 1–2 интеграционных теста с БД (H2/Testcontainers).
- [ ] Простая авторизация: регистрация/логин и защищенный эндпоинт, который без токена/сессии возвращает 401/403.
- [ ] Для списков есть базовая фильтрация/пагинация или лимит.
- [ ] Короткая демо-роль: предзаполненный «демо-пользователь» для быстрого входа.
