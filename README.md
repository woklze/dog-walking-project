## Dog Walking Platform
Минимально рабочая версия бэкенда для платформы выгула собак. Реализована на Java Spring Boot с PostgreSQL.

# Требования
Java 17 и выше
Maven 3.8+ 
PostgreSQL 14+ (локально или в Docker)
Docker (опционально, для запуска БД в контейнере)

# Запуск локально
Шаги для запуска:
1. Скачать код из репозитория
   
2. Создать базу данных в PostgreSQL (Docker):
CREATE DATABASE dogwalking;

3. Настроить переменные окружения (секреты)
Приложение читает параметры подключения к БД из переменных окружения.
В файле application.properties заданы значения по умолчанию, рекомендуется переопределить их через окружение.

Необходимые переменные:
DB_URL – JDBC URL (по умолчанию jdbc:postgresql://localhost:5432/dogwalking)
DB_USER – пользователь БД (по умолчанию postgres)
DB_PASSWORD – пароль БД (по умолчанию 1234)

Переопределение значений в IntelliJ IDEA:
Run → Edit Configurations → Environment variables
DB_URL=jdbc:postgresql://localhost:5432/dogwalking;DB_USER=postgres;DB_PASSWORD=1234

4. Сборка и запуск приложения
В папке с кодом в терминале через Maven
./mvnw clean package -DskipTests (!скиптестс тк тестов пока нет и из-за переменного окружения в пароле бд выдает ошибку при сборке. 
!можно создать файл application-test.properties и настроить в нём встроенную базу H2, чтобы убрать эту штуку)
Приложение будет доступно по адресу http://localhost:8080.

# Проверка работоспособности
Healthcheck: GET /actuator/health
Показывает статус приложения и соединения с БД. Пример ответа:

json
{
  "status": "UP",
  "components": { "db": { "status": "UP" } }
}

или http://localhost:8080/actuator/health

# Хранение секретов
Все секреты (пароль БД, ключи) задаются через переменные окружения. В коде они не хранятся. В application.properties используются подстановки ${VAR:default}.
# Миграции базы данных
При первом запуске Flyway автоматически создаёт таблицы (users, orders, contracts, ratings) на основе скриптов в src/main/resources/db/migration.
# Логирование
Логи выводятся в консоль с указанием времени, уровня и сообщения. Уровень логирования настраивается в application.properties.
# Обработка ошибок
При неверных данных возвращается 400 Bad Request с описанием ошибки. 
Отсутствующие ресурсы — 404 Not Found. 
Нарушение прав доступа — 403 Forbidden. 
Сервер не возвращает 500 на предсказуемые ошибки пользователя.
# Сохранность данных
Данные сохраняются в PostgreSQL; после перезапуска приложения они не теряются.
